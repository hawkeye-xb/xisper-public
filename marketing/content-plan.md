## 零、策略定位

### 你是谁（个人 IP 定位）

一个在语音 AI 行业工作的程序员，用 Vibe Coding 的方式做了一个自己真正想用的产品——Xisper。

**核心标签**：Vibe Coding / 程序员的品位 / 独立产品

**个人主张**：
> 现在没有什么代码是 AI 写不了的。程序员真正不可替代的，是品位——那些抽象的设计模式、对产品的判断力、对技术方案的取舍。

这句话是你所有内容的底层逻辑。

### Xisper 是什么（产品在个人 IP 中的角色）

Xisper 不是一个要大规模推广的商业产品（还没做收费模块），而是你"品位"的一个具象载体：

- 你在公司做语音相关产品，从客户端视角看到了语音输入的问题和机会
- 你觉得输入法应该纯粹做好精准输入，而不是塞一堆 Agent
- 你用 Vibe Coding 的方式把它做出来了
- 它证明了你的产品判断力和技术选择

### 三层内容结构

```
第一层：Vibe Coding 方法论（引流层，吸引程序员/技术爱好者）
   ↓
第二层：产品思维与设计决策（建立深度，展示品位）
   ↓
第三层：Xisper 产品本身（承接层，种子用户转化）
```

不是直接推产品，而是让人因为认同你这个人 → 好奇你做的东西 → 试用 Xisper。

### 不做什么

- 不开产品专属账号（用现有个人账号）
- 不出镜（用录屏 + 图文）
- 不做大规模投放
- 不做付费转化（Alpha 免费）
- 不拉踩竞品

## 二、小红书内容矩阵

### A. Vibe Coding 系列（引流主力，6-8 篇）

这个系列是流量入口。Vibe Coding 话题本身自带关注度，程序员群体传播力强。

#### A-01 | 开篇：什么是 Vibe Coding
- **标题**：我用 Vibe Coding 做了一个 Mac App，从零到上线
- **关键词**：Vibe Coding、AI 编程、独立开发、程序员
- **内容框架**：
  - 什么是 Vibe Coding（用自然语言描述需求，AI 生成代码，人做决策和设计）
  - 我做了什么：一个 macOS 语音输入工具
  - 技术栈选择：Electron + Vue3 + Cloudflare Workers
  - 成果展示：产品截图
  - 引导：后续会分享完整的 Vibe Coding 过程
- **配图**：代码截图 + 产品界面 + AI 对话截图

#### A-02 | Vibe Coding 的工作流
- **标题**：我的 Vibe Coding 工作流：AI 写代码，我做设计决策
- **关键词**：Vibe Coding 工作流、AI 辅助编程、Cursor、Claude
- **内容框架**：
  - 日常开发流程：需求 → 和 AI 对话 → review → 调整 → 迭代
  - 工具链：用了哪些 AI（Cursor / Claude / etc.）
  - 关键心得：AI 能写代码，但架构设计和取舍是人的活
  - 截图展示几个典型的 AI 对话

#### A-03 | AI 写不了的东西
- **标题**：Vibe Coding 一个月后，我发现 AI 真正写不了的是什么
- **关键词**：AI 编程局限、程序员价值、设计模式、技术品位
- **内容框架**：
  - AI 可以写出任何一段代码
  - 但它不知道该不该写、该写在哪、该用什么模式
  - 举几个 Xisper 开发中的实际例子（比如 ASR + LLM 两阶段的架构决策）
  - 结论：程序员的价值在于品位和判断力
- **这篇可能是传播力最强的一篇**

#### A-04 | 技术选型的取舍
- **标题**：独立开发选 Electron 是不是错误？我的真实体验
- **关键词**：Electron、独立开发、技术选型、Mac App
- **内容**：为什么选 Electron、遇到的坑、权衡、最终为什么没后悔

#### A-05 | Vibe Coding 做原生模块

- **关键词**：Rust、napi-rs、原生模块、AI 编程
- **内容**：fn-key-monitor 的开发故事，AI 辅助写 Rust + CGEventTap

#### A-06 | Cloudflare 全家桶搭后端
- **标题**：一个前端用 Cloudflare Workers 搭了整个后端
- **关键词**：Cloudflare Workers、Serverless、前端做后端、全栈
- **内容**：Workers + D1 + R2 + KV 的实际使用体验，成本，踩坑

#### A-07 | 开发数据盘点
- **标题**：Vibe Coding 开发一个完整 App 的真实数据：XX 次 commit，XX 行代码
- **关键词**：独立开发、Vibe Coding、项目数据

#### A-08 | Vibe Coding 的误区
- **标题**：Vibe Coding 不是"让 AI 写完所有代码"
- **关键词**：Vibe Coding 误区、AI 编程、程序员

## 七、内容审核原则

1. **个人视角叙事**：用"我"的口吻，不用"我们团队"
2. **不暴露公司细节**：可以说"在语音 AI 行业工作"，不提公司名和具体产品
3. **不点名竞品**：用"市面上的方案"、"其他工具"替代
4. **真实不装**：Alpha 就是 Alpha，收费没做就说没做，这反而是种子用户愿意参与的理由
5. **不夸大效果**：识别精准是方向，不承诺完美
6. **行业角色而非纠错包**：面向用户统一用"行业角色"
7. **Vibe Coding 不是噱头**：分享真实经验，不鼓吹"AI 替代程序员"
