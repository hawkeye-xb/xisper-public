# 3.1 零配置 — 不需要 API Key

> 软性差异化。不点名任何竞品。只展示 Xisper 有多简单。

## 时长：60秒

## 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-8秒 | 屏幕文字："用语音输入，需要几步？" | |
| 8-18秒 | 展示一个通用的"API Key 配置"画面（匿名化，不带竞品 logo）：API Key 输入框、Secret、区域选择、模型选择等等 | "有些工具要你去申请 API Key、选服务商、填密钥、选模型……" |
| 18-25秒 | 放大复杂度：不同服务商功能不同，有的只支持通用热词，有的支持对话级热词，还有文档要看 | "而且每家接口不一样，效果也不一样。光是搞明白这些就够头疼的。" |
| 25-35秒 | 硬切到 Xisper：下载 → 打开 → 按 Fn → 说话 → 文字出来 | "或者：下载、打开、按 Fn、说话。" |
| 35-48秒 | 展示设置页面——干净、简洁。没有任何 API Key 输入框。只有用户偏好：快捷键、声音、热词 | "没有 API Key。没有服务商选择。我们在后台根据你的网络环境，自动路由到最优的语音识别服务。" |
| 48-55秒 | 快速蒙太奇：在三个不同 App 里 Fn 听写（每个 1 秒） | "你只管说。剩下的我们搞定。" |
| 55-65秒 | Logo + "免费下载" | "Xisper。按 Fn 就行。" |

## 注意事项

- 关键：不要展示任何竞品的 UI 或名字。需要的话用通用/打码的配置页面，或者纯口述
- 对比的是"复杂配置" vs "零配置"——让观众自己下结论
- 语气尊重——"有些工具"就够了，不点名
