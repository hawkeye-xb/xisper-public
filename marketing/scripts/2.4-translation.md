# 2.4 翻译模式 — 跨语言沟通

> 目标用户：需要用非母语写作的人（比如中文母语但要写英文邮件）。

## 时长：90秒

## 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-8秒 | Gmail 写邮件窗口，空白邮件，收件人是一个外国同事 | "要写英文邮件，但脑子里全是中文？" |
| 8-18秒 | 展示 Xisper 设置 → 翻译目标语言设为"English" | "设一次目标语言就行。" |
| 18-35秒 | 在邮件正文里，按 Fn+T，说："你好 Mike，上次会议讨论的方案我已经整理好了，附件里是最新版本的设计文档，麻烦你看一下，有问题随时找我" → 流式出现中文识别 | "按 Fn+T，用你的母语说。" |
| 35-48秒 | 松手 → 英文翻译出现在邮件里："Hi Mike, I've compiled the proposal we discussed in our last meeting. The latest version of the design document is attached. Please review and let me know if you have any questions." | "地道的商务英文——不是机翻味。" |
| 48-60秒 | 快速演示：在 Slack 里 Fn+T，说中文 → 英文输出 | "哪里都能用。Slack、文档、任何输入框。" |
| 60-75秒 | 反向演示：目标语言设成中文，打开微信，Fn+T，说英文 → 中文输出 | "反过来也行——英文说，中文出。" |
| 75-82秒 | 展示历史记录：原话和翻译都保存了 | "原文和译文都保存在历史里，随时可查。" |
| 82-90秒 | Logo + 标语 | "Xisper 翻译。用你的语言想，用他的语言写。" |

## 注意事项

- 翻译质量是卖点——用真实的商务邮件内容，不要用太简单的句子
- Fn+T 的触发方式要清晰展示（和普通 Fn 听写区分开）
- 展示流式效果：先看到中文被识别出来，然后翻译结果出现
