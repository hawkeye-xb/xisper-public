# 2.1 开发者 — 中英混说的精准识别

> 核心卖点：中国开发者说英文术语发音千奇百怪，普通 ASR 全军覆没，Xisper 纠错包专治这个。

## 时长：90秒

## 真实痛点

中国开发者说英文术语的常见问题：
- "dev" 说成 "戴夫/得五/呆服"
- "staging" 说成 "死得金/斯泰金"
- "merge" 说成 "默基/莫基"
- "rebase" 说成 "瑞贝斯"
- "schema" 说成 "斯个马"
- "async" 说成 "阿辛克"
- "Vue" 说成 "维优"
- "Vite" 说成 "微特/位特"
- "webpack" 说成 "为拍克"
- "localhost" 说成 "楼口后死特"
- "refactor" 说成 "瑞法克特"
- "debug" 说成 "迪巴格/底八个"

这些词天天用，但中国人发音就是不标准，普通语音识别根本听不懂。

## 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-8秒 | 黑底白字标题："中国程序员说英文，语音识别能听懂吗？" | |
| 8-25秒 | **不开纠错包**，对着 Xisper 快速说一段中英混杂的话："我把这个 feature 从 dev 分支 merge 到 staging，然后跑一下 webpack 的 build，看看有没有 async 的问题，schema 也要改一下" → 展示识别结果，一堆乱七八糟的中文音译 | 字幕提示："关闭纠错包的效果 👆"（让用户看到"戴夫"、"默基"、"死得金"之类的） |
| 25-32秒 | 打开热词页面 → 开启「程序员」纠错包 → 快速闪一下里面的规则列表 | "打开程序员纠错包。40 多条规则，专门治中国人说英文。" |
| 32-50秒 | **开着纠错包**，同样的话再说一遍 → 这次 dev、merge、staging、webpack、async、schema 全部精准 | "dev、merge、staging、webpack、async、schema——口音再重也认得出来。" |
| 50-62秒 | 继续说一段更实际的："把 prod 环境的 debug 日志关了，localhost 的 endpoint 要改成线上的，Vue 组件需要 refactor 一下" → 全部精准 | "prod、debug、localhost、endpoint、Vue、refactor——你怎么发音都行，它都听得懂。" |
| 62-75秒 | 在 Cursor 里演示实际用途：光标在函数上方，Fn 说一段注释 → 中文说出来，英文注释写进去 | "说中文，写英文注释。它知道你在代码编辑器里。" |
| 75-85秒 | 快速蒙太奇：commit message、PR 描述、Slack 开发频道 | "注释、commit、PR、团队沟通——开口就行。" |
| 85-90秒 | Logo + 标语 | "Xisper。口音不标准？没关系。" |

## 录制要点

- **关键：前后对比要震撼**。关闭纠错包时故意用中式发音说那些术语，让识别结果尽量难看
- 开启纠错包后同样的发音，结果精准——这个反差就是卖点
- 不用刻意念标准英语。就用你平时开会时候的发音，越真实越好
- 建议录之前先测试一遍，确认哪些词的对比效果最明显，重点说那几个
