# 2.3 Webhook — 会议纪要同步到 Notion/飞书

> 目标用户：需要做会议记录、并且希望记录自动同步到某个平台的人。

## 时长：90秒

## 录前准备

- 一个可用的 Webhook 端点（Notion、飞书机器人，或用 requestbin 演示）
- 在 Xisper 设置里提前配好 Webhook URL

## 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-8秒 | "会议进行中"的画面（背景放一个视频会议窗口模拟） | "开会了。不用手忙脚乱记笔记。" |
| 8-18秒 | 展示 Xisper 设置 → Webhook 部分 → 已配好一个 URL（比如 Notion API） | "提前设置一次：把 Xisper 指向你的 Notion、飞书或任何 Webhook。" |
| 18-35秒 | 按 Fn，说："刚才讨论了三个议题，第一是新版本的发布时间定在下周三，第二是需要增加一个用户反馈入口，第三是性能优化的优先级提到 P1" → 文字流式出现 | "直接说。Xisper 实时转写并润色。" |
| 35-45秒 | 松手 → 润色后的会议纪要出来。历史记录里显示 Webhook 已触发（绿色对勾） | "说完了。已经自动发到 Notion 了。" |
| 45-58秒 | 切到 Notion（或飞书）→ 展示刚刚收到的内容，格式整齐 | "打开 Notion 看——已经在了。不用复制粘贴。" |
| 58-72秒 | 再说一条简短的补充。展示追加到同一个 Notion 页面或飞书消息 | "每次听写都自动同步。说着说着，知识库就建起来了。" |
| 72-82秒 | 展示历史记录页面：所有转写 + Webhook 发送状态 | "完整历史记录，还能回放音频，随时核查。" |
| 82-90秒 | Logo + 标语 | "Xisper + Webhook。说一次，到处有。" |

## 注意事项

- "魔法时刻"是在 Notion/飞书里看到刚说的话秒级出现
- Webhook 配置画面一闪而过就好，不要让它看起来很复杂
- 提一句：任何支持 Webhook 的平台都能用，不只是 Notion
